# Vision Dine: Technical Roadmap (Iterate '26)

## Phase 1: Infrastructure & Data Integrity
- [ ] **Absolute URL Logic:** Update the dish fetching logic to use absolute Supabase URLs for `model_3d_url`.
- [ ] **CORS Resilience:** Add error handling for 3D model fetches; implement a local `/public/models/default.glb` fallback.
- [ ] **Database Mapping:** Ensure `scale_factor` (float8) is correctly retrieved and defaulted to 1.0 if null.

## Phase 2: VisionScale™ Admin Logic
- [ ] **Dimension Extraction:** Implement `onLoad` handlers to grab model dimensions via `getDimensions()`.
- [ ] **Real-Time Calculation:** Implement the scaling formula:
  $$Size_{cm} = Base_{meters} \times Scale\_Factor \times 100$$
- [ ] **Slider Binding:** Bind the UI slider to the scale state, updating the "Estimated Real-World Size" text in real-time.

## Phase 3: Premium AR Experience (Mobile)
- [ ] **The "Launch" Trigger:** Build a full-screen "TAP TO ENTER AR" overlay to satisfy user-gesture requirements for camera access.
- [ ] **Transparent Layers:** Force all parent containers of <model-viewer> to `bg-transparent !important` to ensure the camera feed is visible.
- [ ] **UX Polish:** Implement the Top Pill (Dish Name), Center Info Card, and Bottom Shutter Bar using backdrop-blur-xl and Gold accents.

## Phase 4: The Viral Loop (Local Capture)

- [ ] **Watermark Engine:** Programmatically overlay the Vision Dine logo onto the captured blob.
- [ ] **Local Save:** Implement a temporary <a> tag trigger to save the image directly to the phone's "Downloads" folder (No Supabase upload).
- [ ] **MediaRecorder (Video):** (Stretch Goal) Implement long-press canvas recording for social sharing.

## Phase 5: Deployment & Stability
- [ ] **Vercel Build:** Final pass on all types and environment variables.
